# Byzy 校服系统全量交付报告与使用手册 (V1 + V2)

## 1. 项目交付概览
Byzy 校服系统是一款为学校和家长量身定制的垂直电商解决方案。历经 V1 的基础搭建与 V2.0.0 的深度功能升级，目前已形成一个具备高可靠性、高可追溯性的成熟平台。

## 2. 系统核心模块 (Module Summary)
1. **组织管理**: 学校、年级、班级三级架构。
2. **产品管理**: 季度产品隔离、单价配置、可视化上架。
3. **订购流**: 姓名+手机+生日验证、支付暂存、成功固化、自动分班。
4. **售后中心**: 调换、部分退款、申请撤回、审批流。
5. **物流中心**: 批量发货、状态追踪、物理快照存证、对账单导出。
6. **统计日报**: 实时看板、分利润汇总、历史波动追溯。

## 3. 使用手册 - 管理员权限版

### 3.1 基础架构搭建
- **第一步**: 进入“后台管理 -> 架构管理”。
- **第二步**: 添加学校，随后进入该学校面板添加年级。
- **第三步**: 使用“花名册导入”功能上传 Excel。系统会自动将已有订单关联至对应班级。

### 3.2 销售配置
- 在“商品配置”中，管理员可实时开关夏/秋/冬装。建议在正式开售前，检查单价（以分存，元显）是否符合预期。

### 3.3 售后对账
- **由于 V2.0.0 引入了部分退款**，对账时请优先导出 `ShipmentBatch`（历史详情）。它反映的是发货瞬间的真实物流量，不受用户后续取消订单的影响。

## 4. 使用手册 - 技术维护版

### 4.1 核心流程防错
- **支付固化**: 开发者如需修改支付逻辑，需重点关注 `finalizeOrder` 方法。该方法包含事务逻辑，若中途失败，必须抛出错误以防止数据不一致。
- **快照存储**: `ShipmentBatch` 采用 JSON 列存储。严禁在手动修改数据库时破坏 JSON 结构的完整性。

### 4.2 环境依赖
- **Node.js**: 建议版本 18.x 及以上。
- **MySQL**: 建议版本 8.0，需支持 `JSON` 数据类型。

## 5. 手把手部署教程 (Step-by-Step)

### 5.1 数据库初始化
1. 执行 [v2_full_schema_sync.sql](file://<brain_dir>/v2_full_schema_sync.sql) 创建 12 张核心表。
2. **重要**: 确保数据库编码为 `utf8mb4`，以兼容学生姓名中的特殊字符。

### 5.2 服务端 (Server)
1. 进入 `server` 目录，执行 `npm install`。
2. 配置 `.env` 环境，确保连接池数量合理。
3. 创建 `public/uploads` 目录并给予 777 权限。
4. 运行 `npm run dev` 启动 API 枢纽。

### 5.3 管理后台 (Admin) & 家长端 (WebClient)
1. 分别在对应目录执行 `npm install`。
2. 管理端通过 `vite.config.ts` 配置后端 Proxy；家长端通过 `.env.local` 配置后端 URL。
3. 执行 `npm run build` 生成生产包。

## 6. 维护与支持 (FAQ)
- **数据同步**: 若本地与线上结构不一，请务必先备份，再执行 [v2_schema_patch.sql](file://<brain_dir>/v2_schema_patch.sql) 进行无损补丁。
- **性能**: 随着记录增加，建议定期对 `orders` 表的 `student_id` 和 `shipped_at` 创建索引（系统默认已包含基础索引）。

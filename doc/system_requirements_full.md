# Byzy 校服系统全量需求规格说明书 (V1 + V2)

## 1. 项目概述
Byzy 校服系统是一个连接学校、家长与校服供应商的综合性 eCommerce 平台。其核心目标是简化校服选购流程，解决学校组织架构复杂、售后对账混乱、发货历史难以追溯等传统痛点。

## 2. 核心模块与功能

### 2.1 组织架构管理 (Infrastructure)
- **多层级学校架构 (V2)**：支持“学校 -> 年级 -> 班级”的三级管理模型。
- **动态名册 matching (V2)**：支持管理员上传 Excel 花名册，系统自动根据学生三要素（姓名、手机、生日）与已支付订单进行智能匹配。
- **架构灵活调整**：支持在不破坏原有数据关联的情况下，动态增删年级与班级。

### 2.2 产品与配置中心 (Catalog)
- **场景化产品模型**：按季度（夏装、秋装、冬装）及校区维度管理产品。
- **视觉配置 (V2)**：管理员可在后台配置各校的专属图片、单价及上下架状态。
- **辅助配置 (V2)**：支持上传学校专属的尺码对照表，辅助家长选码。

### 2.3 订购与交易流 (Ordering)
- **双通道身份验证**：
  - **V1**: 基于身份证号的强校验。
  - **V2 (推荐)**: 采用“姓名+手机号+生日”联合校验，降低敏感信息填报门槛，同时准确识别重复购买逻辑。
- **订单生命周期管理**：
  - **订单暂存 (V2)**：未支付前仅存入 `OrderTemp`，防止产生虚假 PENDING 订单阻塞对账。
  - **状态驱动**: PENDING (待付) -> PAID (已付) -> SHIPPED (已发货) -> AFTER_SALES (售后中) -> CANCELLED (已取消)。
- **支付集成**: 对接外部支付平台，实现自动 Webhook 回调与订单状态固化。

### 2.4 后台管理工作台 (Admin Workspace)
- **多维度统计看板 (Dashboard)**：
  - **实时金额统计**：按分转化元展示，支持减去退款后的净销额统计。
  - **时间过滤 (V2)**：支持按自定义日期范围查询销售概况。
- **订单中心 (Order Center)**：
  - **精准搜索**：支持姓名、手机号、订单号全词或部分模糊匹配。
  - **补单功能**：管理员可手动为线下转账学生创建补充订单。

### 2.5 售后服务系统 (After-Sales)
- **家长自助手册 (V2)**：家长可在 H5 端针对已付款订单发起“申请调换”或“申请退款”。
- **灵活退款颗粒度 (V2)**：支持整单退款或针对单件商品的部分退款，系统自动扣减库存套数与实收金额。
- **状态联动**：售后申请成功后，订单自动进入售后流程，发货清单同步锁定。

### 2.6 物流与批次存证 (Logistics)
- **确认发货 (V2)**：管理员点击发货时，系统自动记录 `shipped_at` 并同步订单状态。
- **物理快照存证 (V2)**：每次发货生成不可篡改的 JSON 快照（`ShipmentBatch`），确保无论后续如何售后，原始发货单据永远可供追溯。
- **对账单导出**：支持“一人一行”模式导出，聚合展示学生所有季度的最终发货尺码与明细。

## 3. 用户角色定义
- **系统管理员**：管理学校架构、配置单价、处理售后审批、导出清单。
- **家长用户**：填写学生信息、选购尺码、在线支付、查询订单、自助申请售后。
- **仓库/财务人员**：根据系统生成的批次清单进行实物打包与资金对账。
